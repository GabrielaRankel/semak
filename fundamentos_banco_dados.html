O que é um SGBD? Tipos e APLICAÇÕES?

Um SGBD (Sistema de Gerenciamento de Banco de Dados) está definido como um conjunto de programas que eficiencia e segura permite criar, gerenciar e manipular bancos de dados. Ele funciona como um meio termo entre o sistema (ou usuário) e os dados almacenados e garante que os dados possam ser acessados, atualizados e organizados sob controle.

Funções principais de um SGBD:
* Armazenar grandes volumes de dados.
* Permitir consultas e atualizações rápidas.
* Garantir segurança e controle de acesso.
* Evitar redundância e inconsistência de informações.
* Fazer backups e restaurações automáticas.

SGBD Relacional (RDBMS)
Organiza os dados em tabelas formadas por linhas e colunas, facilitando a consulta e atualização das informações. Utiliza a linguagem SQL para manipular os dados de forma estruturada e segura.

SGBD NoSQL (Não relacional)
Projetado para lidar com grandes volumes de dados não estruturados e com diferentes formatos. Oferece alta flexibilidade, escalabilidade e desempenho em aplicações modernas.

SGBD Hierárquico
Armazena os dados em uma estrutura de árvore, com relações entre “pai” e “filho”. É eficiente para consultas previsíveis e sistemas que exigem alta velocidade de processamento.

SGBD em Rede
Permite múltiplas relações entre os dados, formando uma rede complexa de conexões. É usado em aplicações que precisam representar relacionamentos mais detalhados entre informações.

SGBD Orientado a Objetos
Baseia-se no conceito de objetos, combinando dados e comportamentos em uma mesma estrutura. Facilita a integração entre bancos de dados e linguagens de programação orientadas a objetos.

O que é um DICIONÁRIO de dados?

Um dicionário de dados é um repositório que descreve e organiza todas as informações sobre os dados de um banco de dados.
Ele mostra como cada dado é estruturado, nomeado, armazenado e relacionado — por exemplo, o tipo de dado (texto, número, data), o tamanho máximo, se pode ser nulo e a qual tabela pertence.
É como um manual técnico do banco de dados, que ajuda desenvolvedores e analistas a entenderem o significado e a função de cada campo dentro do sistema.

1. Numéricos
Usados para armazenar números inteiros ou decimais.
INTEGER / INT → números inteiros (ex.: 1, 25, -10).
SMALLINT / BIGINT → números inteiros pequenos ou muito grandes.
DECIMAL / NUMERIC / FLOAT / DOUBLE → números com casas decimais (ex.: 10.5, 3.1415).

2. Texto / Cadeia de caracteres
Usados para armazenar palavras, frases ou textos curtos.
CHAR(n) → tamanho fixo, ex.: CHAR(5) sempre ocupa 5 caracteres.
VARCHAR(n) → tamanho variável, até n caracteres.
TEXT / CLOB → textos longos, como descrições ou artigos.

3. Data e Hora
Armazenam informações temporais, como datas e horários.
DATE → apenas a data (ex.: 2025-10-14).
TIME → apenas a hora (ex.: 14:30:00).
DATETIME / TIMESTAMP → data e hora juntos.

4. Booleano / Lógico
Armazena valores verdadeiros ou falsos.
BOOLEAN / BOOL → ex.: TRUE / FALSE, 1 / 0.

5. Binários / Arquivos
Usados para armazenar dados não textuais, como imagens, vídeos ou arquivos.
BLOB (Binary Large Object) → grandes objetos binários, ex.: fotos, PDFs.

6. Outros tipos especiais
Alguns bancos de dados oferecem tipos avançados:
ARRAY / JSON / XML → para armazenar coleções ou dados estruturados.
UUID / GUID → identificadores únicos universais.
ENUM → valores pré-definidos, ex.: ('Masculino', 'Feminino' ...).

Modelagem de Banco de Dados: DER e MER

O que é Modelagem de Dados?
É o processo de criar um modelo abstrato que representa a estrutura, relações e restrições dos dados de um sistema.
Modelo DER (Diagrama Entidade-Relacionamento)
Representa graficamente as entidades (objetos do mundo real), seus atributos e os relacionamentos entre elas.
Modelo MER (Modelo Entidade-Relacionamento)
É a representação conceitual do DER, focando na estrutura lógica dos dados sem considerar implementação física.

Entidade
Objeto do mundo real (ex: Cliente, Produto)

Relacionamento
Associação entre entidades (ex: Compra, Pertence)

Atributo
Característica da entidade (ex: Nome, Preço)

Cardinalidade
Quantidade de relacionamentos (1:1, 1:N, N:M)
