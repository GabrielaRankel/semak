2. Conceitos Fundamentais

O que é uma Rede mínima?
Uma rede mínima consiste em pelo menos dois dispositivos conectados, podendo ser dois computadores diretamente ligados por um cabo ou via wireless.

O que é a Internet? O que a compõe?
A Internet é uma rede global de redes interconectadas, composta por:
Provedores de serviços (ISPs)
Backbones de fibra óptica
Data centers e servidores
Dispositivos de usuários finais
Roteadores e switches
Protocolos de comunicação (TCP/IP)

O que é a Web?
A World Wide Web (WWW) é um sistema de informação que opera sobre a Internet, utilizando o protocolo HTTP para acesso a documentos interligados por hyperlinks.

3. Infraestrutura Global: Mapa de Fibra Óptica

O que é o Mapa de Fibra Óptica?
É a representação visual da rede global de cabos submarinos e terrestres que formam o backbone da Internet, conectando continentes e países. Características da Infraestrutura Global:
Mais de 1,3 milhões de km de cabos submarinos Conexões transoceânicas entre todos os continentes Velocidades que chegam a terabits por segundo Infraestrutura crítica para economia global Investimentos bilionários em expansão contínua

TCP/IP é melhor para uso real porque é o modelo implementado na internet.
OSI é melhor para aprendizado porque separa mais funções e deixa tudo mais claro.

6. O modelo OSI (Open Systems Interconnection)

Camada 7 - Aplicação: Interação direta com o usuário e software, como navegadores e e-mails.
Camada 6 - Apresentação: Formatação, compressão e criptografia dos dados.
Camada 5 - Sessão: Gerenciamento das sessões de comunicação entre os dispositivos.
Camada 4 - Transporte: Controle da entrega de dados e correção de erros (TCP, UDP).
Camada 3 - Rede: Roteamento dos pacotes entre diferentes redes (IP).
Camada 2 - Enlace de Dados: Transmissão de quadros de dados entre dispositivos na mesma rede (Ethernet).
Camada 1 - Física: Transmissão física dos dados através do meio de comunicação, como cabos e ondas de rádio.

Um protocolo de comunicação é um conjunto de regras que define como dois dispositivos trocam dados em uma rede. Ele estabelece como a informação será enviada, recebida, interpretada e confirmada, garantindo que todos os equipamentos consigam “falar a mesma língua”. Em resumo: protocolo = regras para comunicação funcionar corretamente entre dispositivos. Exemplos: HTTP, HTTPS, TCP, IP, FTP, SMTP.

8. O que é o protocolo IP V4 ? e o IP V6?

Protocolo IPv4
É a quarta versão do IP (Internet Protocol).
Define como os endereços de dispositivos são representados na rede.
Usa endereços de 32 bits, no formato:
192.168.0.1
Quantidade total: cerca de 4,3 bilhões de endereços.
Está próximo do esgotamento.

Protocolo IPv6
É a sexta versão do IP, criado para substituir o IPv4.
Usa endereços de 128 bits, no formato:
2001:0db8:85a3:0000:0000:8a2e:0370:7334
Quantidade total: aproximadamente 3,4 × 10³⁸ endereços (muito maior que o IPv4).
Mais seguro, mais rápido no roteamento e mais moderno.

10. Como é configurado o IP e o TCP em uma programação backend com node.js?

Configuração de IP e Porta (TCP) em Node.js —
Servidor HTTP


const http = require('http');

const IP = '127.0.0.1';
const PORT = 3000;

const server = http.createServer((req, res) => {
  res.end('Servidor rodando!');
});

server.listen(PORT, IP, () => {
  console.log(`Rodando em http://${IP}:${PORT}`);
});

Configuração de TCP puro
(sem HTTP)


const net = require('net');

const IP = '0.0.0.0';
const PORT = 4000;

const server = net.createServer((socket) => {
  console.log('Cliente conectado');

  socket.write('Conexão TCP estabelecida!');

  socket.on('data', (data) => {
    console.log('Recebido:', data.toString());
  });
});

server.listen(PORT, IP, () => {
  console.log(`Servidor TCP rodando em ${IP}:${PORT}`);
});

Configuração com Express.js


        

const express = require('express');
const app = express();

const IP = '0.0.0.0';
const PORT = 8080;

app.get('/', (req, res) => {
  res.send('API funcionando!');
});

app.listen(PORT, IP, () => {
  console.log(`API em http://${IP}:${PORT}`);
});

Como o IP é escolhido na prática
127.0.0.1 → acessível só localmente
0.0.0.0 → recebe conexões de qualquer IP
IP interno da rede → ex.: 192.168.x.x
IP público → servidores em nuvem

11. O que é o protocolo HTTP? E o HTTPS ?

HTTP
É um protocolo usado para transferir páginas e dados na web.
Define como navegador e servidor trocam informações.
Não possui criptografia.

HTTPS
É a versão segura do HTTP.
Usa criptografia SSL/TLS para proteger os dados enviados e recebidos.
Impede espionagem, roubo de informações e alterações no tráfego.

Resumo:
HTTP = sem segurança
HTTPS = comunicação criptografada e protegida
